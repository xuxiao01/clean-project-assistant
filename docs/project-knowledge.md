# 项目名称
清洁工接单项目

# 项目简介
清洁工接单项目是一个面向清洁服务场景的业务系统，主要用于连接平台管理员、雇主和清洁工，帮助完成清洁订单的发布、接单、执行、状态跟踪和结算管理。

本项目的目标是让清洁工能够更方便地查看可接订单、接取任务、执行清洁工作，并让管理员和雇主能够清晰掌握订单状态和人员安排情况。

# 项目角色

## 1. 管理员
管理员负责平台整体运营和订单管理，主要职责包括：

- 创建和维护清洁项目
- 发布清洁订单
- 分配或审核订单
- 管理清洁工信息
- 查看订单执行状态
- 处理异常订单
- 管理结算信息

## 2. 雇主
雇主是提出清洁需求的用户，主要职责包括：

- 提交清洁需求
- 查看自己发布的订单
- 查看订单执行进度
- 确认服务完成情况
- 对服务结果进行反馈

## 3. 清洁工
清洁工是接单并执行清洁任务的人员，主要职责包括：

- 查看可接订单
- 接取适合自己的订单
- 查看待执行任务
- 到场执行清洁工作
- 更新订单状态
- 查看历史订单和收益记录

# 业务目标
本项目主要解决以下问题：

1. 清洁订单发布后，清洁工无法及时获取任务信息
2. 管理员难以实时掌握订单执行进度
3. 订单状态流转不清晰，容易出现遗漏
4. 清洁工接单、执行、完成的全流程缺少统一管理
5. 服务结果和结算数据缺乏统一记录

# 核心业务流程

## 1. 订单创建流程
订单创建流程如下：

1. 雇主提交清洁需求
2. 管理员审核需求信息
3. 管理员创建清洁订单
4. 系统生成订单编号
5. 订单进入“待接单”状态

## 2. 清洁工接单流程
清洁工接单流程如下：

1. 清洁工登录系统
2. 进入订单大厅
3. 查看当前可接订单列表
4. 选择符合时间和地点要求的订单
5. 点击“立即接单”
6. 系统校验订单是否仍可被接取
7. 接单成功后，订单状态变为“待上门”

## 3. 订单执行流程
订单执行流程如下：

1. 清洁工在任务开始前查看订单详情
2. 按照订单地址和预约时间到达现场
3. 开始执行清洁任务
4. 清洁工将订单状态更新为“服务中”
5. 清洁完成后，清洁工将订单状态更新为“待确认”
6. 雇主或管理员确认服务完成
7. 订单状态变为“已完成”

## 4. 异常处理流程
在订单执行过程中，可能出现以下异常情况：

- 清洁工接单后无法按时上门
- 雇主临时取消订单
- 服务过程中发现额外清洁需求
- 服务地址信息有误
- 订单长时间无人接单

异常处理流程如下：

1. 相关人员提交异常说明
2. 管理员查看异常详情
3. 管理员决定是否取消、重新派单或修改订单信息
4. 系统记录异常处理结果

# 订单状态说明

## 1. 待接单
订单刚创建完成，尚未被任何清洁工接取。

## 2. 待上门
订单已被清洁工成功接单，等待按预约时间上门服务。

## 3. 服务中
清洁工已到达现场并开始执行清洁任务。

## 4. 待确认
清洁工已完成服务，等待雇主或管理员确认。

## 5. 已完成
订单服务已确认结束，进入归档和结算阶段。

## 6. 已取消
订单因用户取消、无人接单或其他原因终止。

## 7. 异常中
订单当前存在异常，需要管理员介入处理。

# 主要页面说明

## 1. 登录页面
登录页面用于用户进入系统。不同角色登录后会进入不同工作页面。

## 2. 订单大厅
订单大厅主要面向清洁工，用于展示当前所有可接订单。页面通常包含以下信息：

- 订单编号
- 服务地址
- 服务类型
- 预约时间
- 预计时长
- 订单价格
- 当前状态

清洁工可以在该页面直接点击“接单”。

## 3. 我的订单页面
我的订单页面主要面向清洁工，用于查看自己已经接取的订单。页面通常分为以下几个栏目：

- 待上门
- 服务中
- 待确认
- 已完成
- 已取消

## 4. 订单详情页面
订单详情页面用于查看某一个订单的完整信息，通常包括：

- 订单编号
- 雇主姓名
- 联系方式
- 服务地址
- 服务类型
- 服务要求
- 预约时间
- 订单金额
- 当前状态
- 备注信息

清洁工可在该页面进行状态更新操作。

## 5. 订单管理页面
订单管理页面主要面向管理员，用于统一查看和处理平台订单。管理员可进行以下操作：

- 创建订单
- 查询订单
- 筛选订单状态
- 查看异常订单
- 处理取消订单
- 查看订单执行进度

## 6. 结算页面
结算页面主要用于查看已完成订单的费用情况，包括：

- 订单金额
- 平台抽成
- 清洁工应得收入
- 结算状态
- 结算时间

当前系统支持的服务类型包括：

- 日常保洁

- 开荒保洁
- 厨房清洁
- 卫生间专项清洁
- 搬家后清洁

# 平台规则

## 1. 接单规则
- 只有处于“待接单”状态的订单才允许接单
- 同一订单同一时间只能被一名清洁工接取
- 清洁工接单后，需要按预约时间完成服务
- 若清洁工连续多次爽约，管理员可限制其接单资格

## 2. 状态更新规则
- 接单成功后，订单状态自动变为“待上门”
- 清洁工到场后可将状态更新为“服务中”
- 服务完成后可将状态更新为“待确认”
- 只有管理员或雇主确认后，订单才可进入“已完成”

## 3. 取消规则
- 雇主可在服务开始前申请取消订单
- 管理员可根据情况直接取消订单
- 已进入“服务中”的订单原则上不能直接取消，需走异常处理流程

## 4. 结算规则
- 只有状态为“已完成”的订单才能进入结算流程
- 取消订单不参与正常结算
- 异常订单需由管理员确认处理结果后再决定是否结算

# 常见问题

## 如何查看可接订单？
清洁工登录系统后，可以进入订单大厅查看当前所有状态为“待接单”的订单，这些订单都可以接取。

## 如何接单？
在订单大厅中选择目标订单，点击“立即接单”按钮即可。若系统校验通过，接单成功后该订单会进入“待上门”状态。

## 如何查看自己已经接取的订单？
清洁工可以进入“我的订单”页面查看自己已接取的订单，并按照不同状态进行筛选。

## 如何更新订单状态？
清洁工进入订单详情页后，可以根据实际执行情况更新订单状态，例如从“待上门”更新为“服务中”，再更新为“待确认”。

## 如何确认订单完成？
当清洁工完成服务后，订单状态会变为“待确认”。雇主或管理员确认无误后，订单状态更新为“已完成”。

## 哪些订单不能接？
已被其他清洁工接单、已取消、已完成或异常中的订单不能再接单。

## 清洁工接单后不能上门怎么办？
清洁工应尽快联系管理员说明情况。管理员会根据实际情况处理订单，可能采取取消订单、重新派单或记录异常等操作。

## 订单长时间没人接怎么办？
若订单长时间无人接单，管理员可以调整订单价格、修改预约时间，或通过人工方式重新分配。

## 如何查看历史完成订单？
清洁工可以在“我的订单”页面中切换到“已完成”栏目查看历史订单。

## 如何查看订单收益？
清洁工可以在结算页面或个人收益页面查看已完成订单对应的收入信息。

# 示例场景

## 场景一：正常接单
1. 管理员创建一个日常保洁订单
2. 订单状态为“待接单”
3. 清洁工张三在订单大厅看到该订单
4. 张三点击“立即接单”
5. 系统接单成功，订单状态变为“待上门”
6. 张三按预约时间上门服务
7. 服务开始后状态变为“服务中”
8. 服务完成后状态变为“待确认”
9. 管理员确认服务完成，订单状态变为“已完成”

## 场景二：订单取消
1. 雇主创建订单后，因时间变化申请取消
2. 管理员审核后同意取消
3. 订单状态变为“已取消”
4. 已取消订单不再允许被清洁工接取

## 场景三：接单异常
1. 清洁工接单后因突发情况无法按时上门
2. 清洁工联系管理员说明情况
3. 管理员将订单标记为“异常中”
4. 管理员重新安排其他清洁工或与雇主协商调整时间
